    • Render modes
      Name Description Render location Interactive
      Static Server Static server-side rendering (static SSR).
      The server produces static HTML that doesn't provide for user interactivity or maintaining Razor component state.      		 Server ❌No
      Interactive Server Interactive server-side rendering (interactive SSR) using Blazor Server.
      Blazor events permit user interactivity and Razor component state is maintained by the Blazor framework.      		 Server ✔️Yes
      Interactive WebAssembly Client-side rendering (CSR) using Blazor WebAssembly†.
      Final HTML markup is generated by the .NET WebAssembly runtime on the client. No HTML for the app's client-generated UI is sent from a server to the client for this type of rendering       		Client ✔️Yes
      Interactive Auto Interactive SSR using Blazor Server initially and then CSR on subsequent visits after the Blazor bundle is downloaded Server, then client ✔️Yes
